## Bug Details

---

### Bug #1 — OTP Send Returns 500 (BLOCKER)

**Severity:** Critical  
**Status:** Open — requires manual action  
**Affects:** Entire login flow (and signup OTP resend)

**Description:**  
`POST /api/auth/send-otp` returns `500 Internal Server Error` for all requests. The UI shows "Failed to send OTP". No OTP step is ever reached.

**Root Cause:**  
The DB migration that adds the `type` column to `otp_tokens` has not been run. The new code inserts `type` and filters by `.eq('type', type)` — both fail because the column doesn't exist in Supabase.

**Fix — run this in Supabase SQL editor:**
```sql
ALTER TABLE otp_tokens ADD COLUMN IF NOT EXISTS type TEXT NOT NULL DEFAULT 'signup';
```

This is backward compatible — existing rows will default to `'signup'`.

---

### Bug #2 — verify-otp Guard Caused React Render Error

**Severity:** Medium  
**Status:** ✅ Fixed in this session  
**File:** `src/app/(auth)/verify-otp/page.tsx`

**Description:**  
Direct navigation to `/verify-otp` (with no signup data) caused two React errors:
- `"Cannot update a component while rendering a different component"`
- Hydration mismatch warning

The guard correctly redirected to `/signup`, but calling `router.replace()` directly in the render body is not allowed by React.

**Fix applied:**  
Moved the redirect inside `useEffect`:
```tsx
useEffect(() => {
  if (!signupData.email) router.replace("/signup");
}, [signupData.email, router]);

if (!signupData.email) return null;
```

### Bug #3 — `/forgot-password` Page is Now Orphaned

**Severity:** Low  
**Status:** Open  
**File:** `src/app/(auth)/forgot-password/page.tsx`

**Description:**  
Since login moved to passwordless OTP, users no longer have passwords to reset. The "Forgot Password?" link was correctly removed from the login page, but the `/forgot-password` route still exists as a live page. It sends a Supabase password reset email which is no longer relevant in the passwordless flow.

The page is not dangerous (no data exposure), but it's confusing if someone navigates to it directly.

**Recommended Fix:**  
Either redirect `/forgot-password` to `/login`, or remove the page entirely since accounts no longer use passwords for login.
